Installation & Integration Guide

This guide explains how to integrate the Mod Controller into your AzerothCore server and how to connect your own modules to the system.

Installation
To install the Mod Controller, follow these steps:

Copy the mod_controller folder into your AzerothCore modules/ directory.

Ensure your CMakeLists.txt is configured to include the src/ directory.

Perform a clean build of your server:

rm -rf build
mkdir build
cd build
cmake ../ -DMODULES=mod_controller
make -j$(nproc)

Integrating Your Modules
Once installed, your modules can communicate via the central Postman system.

A. Registering as a Subscriber
If your module needs to react to specific events, register it with the Controller during the startup phase.

// Register module (ID: 100) to listen for event (ID: 500)
sController->PostmanRecipient(100, 500);

B. Dispatching Events
To broadcast an event to all subscribed modules, use the Postman method:

// Dispatching event (ID: 500) from module (ID: 100) targeting a specific Guid
sController->Postman(100, 500, targetGuid);

C. Security Validation
The system includes a PostmanExaminer to prevent unauthorized event dispatching. You can implement custom logic inside Controller.cpp to validate sender permissions:

// The Examiner automatically triggers upon Postman calls
sController->PostmanExaminer(senderId, eventId);

Recommended Best Practices

Event IDs: Use a shared constant header file (e.g., EventConstants.h) across your modules to prevent ID collisions.

Singleton Usage: The controller is accessible globally via the sController macro, provided you have defined it in your Controller.h.

Clean Builds: Always perform a clean build after making changes to the module structure to ensure the ModulesScripts.cpp file is correctly regenerated by CMake.
